Core pipeline

From a scanned page to a live listing

A flatbed scan holds nine cards in a 3×3 grid. The pipeline turns each page into individual, identified, priced, deduped eBay listings without manual data entry.

Scan→ Grid split→ Identify (agents)→ Dedup guard→ Price→ Post to eBay
1

Scan

Cards are scanned nine at a time. One image, nine cards.

2

Grid split

A forced-grid splitter cuts each page into nine card images, snapping the cut lines to the whitest bands between cards so full-bleed and die-cut cards don't get clipped.

3

Identify — multi-agent fan-out

One agent per scan reads the nine card images and returns a structured record for each: year, brand, parallel/insert, player, team, rookie flag, numbered flag, and a clean eBay title. Seventeen scans are read in parallel, not one at a time.

parallel subagents · vision · structured output
4

Dedup guard — oversell prevention

Before anything posts, every candidate is checked against three sets: cards already listed, cards sold in the last 90 days, and duplicates within the same batch. A numbered card you own one copy of can never end up listed twice. This guard was added after a double-sale and has held clean across 300+ posts.

sold-aware · token similarity
5

Price

Each card is priced against SportsCardsPro with a confidence gate and a sanity cap, falling back to a velocity tier when no confident match exists. Autos and rare parallels are routed to a separate deep-dive that verifies the exact card and pulls comparable sales before pricing.

multi-source · confidence-gated
6

Post to eBay

Cards are listed through the eBay Trading API with image upload, returns-not-accepted, and a per-card dupe check at post time. The snapshot is refreshed immediately before every run so the dedup decision is always made against current state.

Daily automation

The morning routine

About 25 agents run each morning. Every agent produces a plan and a dashboard first (a dry run); the real-world actions that change live listings are applied in a safe order only after review.

Repricing

Moves prices toward market, capped per run for safety. Built for velocity — prices trend down to move inventory.

Markdowns

Age-tiered markdowns on stale stock, floor-protected so a card never sells below its sold-history floor.

Promoted listings

Pushes per-listing ad bids across the catalog — the visibility lever for a low-traffic store.

Offers to watchers

Sends targeted discounts to the highest-intent buyers: people already watching a card.

Best Offer

Enables and re-clamps accept/decline thresholds so a price drop never breaks an existing offer rule.

Health & digests

Inventory, photo audit, price consistency, Cassini visibility scoring, and a daily digest — all rendered as dashboards.

Intelligence layer

What makes it more than a script

The pipeline is wrapped in agent panels that handle the judgment calls a solo seller would otherwise make by hand.

Sourcing committee

A panel of agents with distinct personas (comp analyst, trend scout, value investor, velocity flipper, risk manager) debates and ranks which players to buy next, scored against current inventory depth.

Three-card lots

A four-expert committee bundles slow, low-view singles into themed lots priced to move, then the originals are delisted once a lot sells.

Premium-card deep-dive

Autos, serial-numbered cards, and rare die-cut parallels are pulled out of the velocity flow and individually verified and comp-priced.

Self-auditing code review

Code changes are reviewed by a fan-out of agents across nine angles; each finding is independently verified before it's accepted, then the fixes are re-reviewed.

QC review page

Every posted card is re-checked against its image with a confidence score, surfacing likely mis-identifications before a buyer finds them.

Analytics dashboard

Listing and sales titles are parsed into structured fields, giving filterable insight into top players, brands, parallels, and what is actually selling.

How the agents work

Fan out, verify, synthesize

The heavy work is done by deterministic workflows that orchestrate many short-lived agents rather than one long conversation.

  • Fan-out: independent work (identifying 160 cards, reviewing nine code angles) runs as parallel subagents, so wall-clock time is the slowest single task, not the sum.
  • Structured output: agents return validated JSON, not prose — titles, prices, verdicts — so results drop straight into the pipeline.
  • Adversarial verification: findings and identifications are confirmed by separate agents before they're trusted, which is how mis-IDs and false bug reports get filtered out.
  • Committees: for judgment calls, several agents argue distinct positions and a chair synthesizes the decision.
